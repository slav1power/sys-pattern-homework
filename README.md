# Домашнее задание к занятию "`Репликация и масштабирование. Часть 2`" - `Уляшев Дмитрий`



### Задание 1

1.1 Активный Master-сервер и пассивный репликационный Slave-сервер:
 - Резервное копирование в реальном времени - данные постоянно реплицируются
 - Быстрое восстановление - при падении Master можно быстро переключиться на Slave
 - Минимизация downtime - сокращение времени простоя системы
 - Снижение нагрузки на Master - часть запросов на чтение можно перенаправить
 - Безопасное выполнение бэкапов - можно делать на Slave без влияния на продакшн

1.2 Master-сервер и несколько slave-серверов:
 - Линейное масштабирование - каждый новый Slave увеличивает скорость чтения
 - Распределение нагрузки - запросы на чтение распределяются между несколькими Slave
 - Географическое распределение - Slave'ы могут быть в разных датацентрах
 - Репликация цепочкой - Slave'ы могут реплицировать с других Slave'ов
 - Множественные копии данных - снижает риск потери данных
